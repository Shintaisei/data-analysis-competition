# 1位解法との対応と改良の余地（全項目）

atmaCup #15 1位解法でやっていたことと当コンペでの対応状況を整理し、**まだ試していないもの・実装で改良の余地があるもの**をすべて挙げる。ベースラインは **Public 0.76479**（`submission_2hop_bpr64_only.csv`）。

---

## 1. 1位解法でやっていたこと → 当コンペでの対応

| 1位の内容 | 当コンペでの対応 | 状況 |
|-----------|------------------|------|
| **Seen / Unseen 二モデル** | 批評家基準で seen 用・unseen 用の 2 本。CV = seen×0.77 + unseen×0.23。 | **不要**（EDA で test の批評家は 100% train に登場・unseen=0）。08 §6 参照。 |
| **接続情報の活用（implicit）** | train+test の (user, item) を結合し、implicit で埋め込みを特徴に。 | **実施済み**。BPR/ALS で批評家・映画ベクトルを 55 特徴に追加。BPR 64 で 0.76479。 |
| **「集約前の値 / ユーザーごとの集約(平均)」** | 値 / 批評家ごとの平均。好みからどれだけ離れているかを表す。 | **実施済み**。`run_atmacup_ratio` で複数パターン試したが、単体では 0.753〜0.754 台。implicit ほどは伸びず。 |
| **2-hop 集約** | user ごと集約 → そのアイテムを視聴したユーザーたちの集約をさらに集約。 | **試したが当コンペではスコア低下**。映画側「その映画をレビューした批評家」の Fresh 率平均・critic_te 平均・人数を足すと 0.72 台。target リーク等が疑われている。08 §2.5。 |
| **その anime と類似した作品をその user がどれだけ見ているか（編集距離の統計）** | 1位では「特徴量的に一番効いていそう」と記載。 | **未実施**。当コンペでは「その映画に類似した映画をその批評家が何本レビューしているか」に相当。類似度はメタやタイトル等で定義可能。 |
| **user ごと groupby → anime ごと groupby** | anime のメタ（members, watching 等）を user で集約し、さらに anime で集約して 2-hop 特徴に。 | **一部のみ**。当コンペでは「映画をレビューした批評家の統計」は 2-hop で試したが落ちた。「批評家ごとに映画メタを集約 → 映画ごとにその集約をさらに集約」は未実施。 |
| **LightGBM** | メインの GBDT。 | **使用中**。単体で最良。 |
| **Neural Network** | アンサンブル用に NN を 1 本。 | **未実施**。当コンペでは LGB のみで提出。07 に「多モデル+Ridge」はあるが 55 特徴ベース。BPR 64 ベースで NN は未。 |
| **Surprise（Explicit Feedback 用）** | SVD, SVD++, KNN 等。1位はスコア予測コンペなので Explicit が使えた。 | **未実施**。当コンペは Fresh/Rotten の二値。レーティング予測を 1 特徴として使うことは可能だが未実装。 |
| **LightGCN** | 推薦用 GNN。接続情報のみで学習。unseen ユーザーには使えない。 | **未実施**。PyTorch Geometric 等での実装が必要。 |
| **独自 GNN（Heterogeneous）** | SAGEConv + LayerNorm、入力を保持して過剰平滑化対策。 | **未実施**。 |
| **スタッキング（LinearRegression / Ridge）** | 複数モデルの予測を Ridge でメタ学習。 | **07 で LGB/XGB/CatBoost + Ridge は実装済み**。38 特徴・55 特徴ベースで実施。**BPR 64 ベース（現ベースライン）では未実施**。 |
| **Multi hot + SVD** | genres, producers 等を multi hot して SVD で次元圧縮。 | 当コンペでは genre_* 等はカテゴリとして利用。**SVD で圧縮した 1 塊を特徴に**は 05/06 等で別軸。1位風の「user ごと集約した SVD 特徴」は未。 |
| **Target encoding** | カテゴリの TE。 | **実施済み**（critic_name_te_ts 等）。1位のように多様な TE の組み合わせや「割り算と組み合わせ」は一部のみ。 |
| **特徴量管理（feather）・実験管理（wandb, hydra）** | 前処理結果の保存、パラメータ管理。 | 当コンペでは未整備。実装の改良余地（インフラ）。 |

## 2. 実装で改良の余地があるもの（全項目）

以下は **すべて**「まだ試していない」または「実装はあるが現ベースライン（BPR 64）に載せていない」ものを列挙する。優先度の目安は「伸びそうな改善」は `docs/09_IMPROVEMENT_NEXT.md` を参照。

### 2.1 協調フィルタ・接続情報（08 系）

| 項目 | 内容 | 現状 | コスト |
|------|------|------|--------|
| ALS factors=32 / 64 | BPR と目的関数が違うので ALS の次元を上げて 1 本。 | ALS 16 のみ実施。 | 低 |
| BPR 64 + movie_review_count 1 列 | 2-hop のうちリークなしのレビュー数だけ。bpr_count は BPR 16 で微増の実績。 | 未実施（BPR 64 ベースでは）。 | 低 |
| BPR 64 と ALS 64 のブレンド | 提出 2 本の予測を加重平均。 | 未実施。 | 低 |
| BPR 64 と BPR 128 のブレンド | 同上。 | 未実施。 | 低 |
| confidence 重み（ALS） | 接続を 0/1 ではなくレビュー数等で重み付け。implicit の ALS は対応。 | 行列は常に 0/1。 | 中 |
| NMF / TruncatedSVD（CF 行列） | 同じ (critic, movie) 行列を sklearn の NMF や SVD で分解し、批評家・映画ベクトルを特徴に。 | BPR/ALS のみ。 | 中 |
| ALS のハイパーパラメータ | regularization, iterations を BPR と同様に als_kwargs で渡す。 | デフォルトのみ。 | 中 |
| 時系列で行列を切る | レビュー日で「過去のみ」の接続で行列を作り、リークを減らす。 | 全期間で 1 行列。 | 高 |

## 3. まとめ：優先して試すなら

- **すぐ試せる（コード変更ほぼ不要）**: ALS 64、BPR 64 + movie_review_count 1 列、BPR 64 と ALS 64 / BPR 128 のブレンド（09 に記載）。
- **1位で効いていたが未実施**: 「その映画に類似した映画をその批評家が何本レビューしているか」、批評家→映画の**メタの 2-hop**（target を使わない）、割り算+2-hop 1 列の組み合わせ。
- **モデル多様化**: BPR 64 ベースで LGB/XGB/CatBoost + Ridge スタッキング、シード平均、重み付きブレンド。
- **実装はあるが BPR 64 に載せていない**: 類似度 TE（02）、擬似ラベル（01）の閾値見直し、scale_pos_weight、特徴選択、CV 加重。

詳細な「伸びそうな改善」の優先度は `docs/09_IMPROVEMENT_NEXT.md`、協調フィルタの詳細は `docs/08_COLLABORATIVE_FILTERING.md`、改善候補の実装検証は `docs/07_IMPROVEMENT_CANDIDATES.md` を参照。
