# Embedding 次元削減・追加実験の技術解説

`train_embedding_submission.ipynb` で使っている**次元削減 8 手法**と**追加実験 4 パターン**の特徴・数理・使い分けをまとめる。技術を選ぶときの参照用。

---

## 1. 次元削減 8 手法の概要

いずれも **train だけで fit し、train / test を transform** する（リーク防止）。embedding は高次元（例: 1536）なので、8〜24 次元に圧縮してから LightGBM の特徴量にする。

| 手法 | 線形/非線形 | 主な仮定・目的 | 計算コスト |
|------|-------------|----------------|------------|
| PCA | 線形 | 分散を最大に保つ直交射影 | 小 |
| TruncatedSVD | 線形 |  Frobenius ノルムで最良近似（中心化なし） | 小 |
| ICA | 線形 | 独立成分の抽出 | 中 |
| Random Projection | 線形 | ランダム射影で距離を近似保存 | 小 |
| Sparse PCA | 線形 | 主成分をスパースに | 大 |
| NMF | 線形（非負） | 非負行列分解 | 中 |
| Kernel PCA | 非線形 | カーネル空間で PCA | 大 |
| UMAP | 非線形 | 多様体の近傍構造を保つ低次元埋め込み | 大 |

---

## 2. 各手法の詳細

### 2.1 PCA（Principal Component Analysis）

- **何をしているか**: データを**分散が大きい方向**に直交射影する。共分散行列の固有ベクトル（主成分）を求め、上位 \(k\) 個で再構成する。
- **数理**: データを中心化した行列 \(X\) に対し、\(X = U \Sigma V^\top\)（SVD）の \(V\) の左から \(k\) 列を取った射影。射影後の分散の和を最大化している。
- **長所**: 解が一意で安定、実装が簡単、解釈しやすい（主成分は「ばらつきの方向」）。
- **短所**: 線形のみ。外れ値にやや敏感（共分散で効く）。
- **embedding での使いどころ**: デフォルト選択。OpenAI 埋め込みのような高次元ベクトルを 8〜32 次元に落とすのに広く使われる。本実験では **PCA 16 が Public 最高（0.75449）**。

---

### 2.2 TruncatedSVD

- **何をしているか**: 行列の**低ランク近似**。SVD の上位 \(k\) 個の特異値・特異ベクトルだけ使う。**中心化しない**。
- **数理**: \(X \approx U_k \Sigma_k V_k^\top\)。Frobenius ノルム \(\|X - X_k\|_F\) を最小にする \(k\) ランク近似。
- **長所**: 中心化不要なのでスパース行列や既に正規化されたベクトルに向く。大規模では Randomized SVD で高速化しやすい。
- **短所**: データの「中心」を考慮しないので、平均が 0 でないと PCA と違う結果になる。
- **embedding での使いどころ**: 埋め込みが正規化済みやスパース寄りのとき、PCA の代替として試す価値あり。

---

### 2.3 ICA（Independent Component Analysis）

- **何をしているか**: 観測を**統計的に独立な成分**の線形混合として分離する。ガウスでない成分を拾う。
- **数理**: \(x = A s\) の \(s\)（独立成分）を推定。FastICA は negentropy などの非ガウス性を最大化して分離。
- **長所**: 独立な「源」に分解できる。ノイズとシグナルが独立ならノイズを 1 成分にまとめやすい。
- **短所**: 収束しないことがある（反復数・初期値に依存）。順序・スケールは一意でない。
- **embedding での使いどころ**: テキスト埋め込みの「トピックっぽい方向」を分離したいときに試す。不安定なら max_iter 増やしたり、外す選択もあり。

---

### 2.4 Gaussian Random Projection

- **何をしているか**: **ランダムな射影行列**で高次元→低次元に写す。行列要素は正規分布などからサンプル。
- **数理**: Johnson–Lindenstrauss の定理：適当な \(k\) なら、距離がほぼ保たれる射影がランダムで存在する。実装はその射影を 1 本引く。
- **長所**: 学習なしで超高速。実装が簡単。次元だけ決めればよい。
- **短所**: 毎回ランダムなので再現性は seed に依存。データの構造は使わないので、PCA より情報は落ちやすいことが多い。
- **embedding での使いどころ**: ベースライン比較や「とりあえず次元だけ減らす」用。コンペでは PCA/SVD の方が効くことが多い。

---

### 2.5 Sparse PCA

- **何をしているか**: 主成分の**荷重（ローディング）をスパース**にする制約付き PCA。各成分が少数の元の変数のみに依存。
- **数理**: 分散最大化に L1 正則化（またはスパース制約）を加え、反復で解く。
- **長所**: 解釈しやすい（どの次元が効いているかがはっきりする）。次元の呪いをスパース性で緩和。
- **短所**: 計算が重い。反復で局所解に陥る可能性。
- **embedding での使いどころ**: 1536 次元のうち「どの軸が効くか」を絞りたいときに試す。計算時間と相談。

---

### 2.6 NMF（Non-negative Matrix Factorization）

- **何をしているか**: 非負行列 \(X\) を \(X \approx W H\)（\(W, H \geq 0\)）と分解する。
- **数理**: 乗法更新などで \(\|X - WH\|\) を最小化。部分的な加算の組み合わせでデータを表現する。
- **長所**: 非負なので「足し算で構成される」解釈がしやすい。スパースで部分的な構造が出やすい。
- **短所**: **入力が非負である必要がある**。本実験では embedding を min-max で [0, 1] にスケールしてから適用。train の min/max で test も同じスケールにしている。
- **embedding での使いどころ**: 埋め込みを非負化できる場合の選択肢。正負の意味が重要な場合は PCA/SVD の方が自然なことが多い。

---

### 2.7 Kernel PCA

- **何をしているか**: データを**カーネル関数で写した空間（特徴空間）**で PCA をやる。非線形な構造を捉えられる。
- **数理**: カーネル行列 \(K\) を中心化したあと、その固有ベクトルで元の点を写す。RBF カーネル \(k(x,y) = \exp(-\gamma\|x-y\|^2)\) がよく使われる。
- **長所**: 非線形。曲がった多様体やクラスタ構造を低次元に反映できる。
- **短所**: サンプル数 \(n\) に対して \(O(n^2)\) 以上。transform は「学習点とのカーネル」で写すので、test が train とずれると不安定になりうる。
- **embedding での使いどころ**: 埋め込みが明らかに非線形な構造を持っていそうなときに 1 本試す。gamma は本実装では `1/次元数` にしている。

---

### 2.8 UMAP（Uniform Manifold Approximation and Projection）

- **何をしているか**: **多様体**を仮定し、近傍関係を保ちながら低次元に埋め込む。t-SNE より距離・構造を保ちやすく、transform で新規点も写せる。
- **数理**: 高次元で k-NN グラフを作り、低次元でそれに合う配置を交叉エントロピーなどで最適化。メトリックは本実装で euclidean（NaN を避けるため）。
- **長所**: 非線形で、局所構造をよく保つ。コンペでもよく使われる。
- **短所**: 計算が重い。transform は「学習データに対する相対的な位置」なので、test が train と分布が違うと信用できなくなることがある。
- **embedding での使いどころ**: PCA/TruncatedSVD のあとに「非線形で効くか」を 1 本試す用。`umap-learn` が必要。

## 3. 追加実験 4 パターンの技術メモ

### 3.1 seed_avg（シードアベレージング）

- **何をしているか**: 同じモデル（movie_title_info + PCA 16 + LGB）を **random_state=42, 43, 44** で 3 本学習し、test 予測を**算術平均**する。
- **効果**: モデルの分散を下げて、スコアを安定・微増させることが多い。コンペの定番。
- **注意**: 学習時間は約 3 倍。提出は 1 本。

### 3.2 pca24（次元数を 24 にした PCA）

- **何をしているか**: 手法は PCA のまま、**次元数だけ 24** にした 1 本。
- **効果**: 16 が良かったので「少し情報を増やす」と効くかどうかを検証。多すぎると過学習のリスクもある。

### 3.3 ensemble_two（2 本の提出の平均）

- **何をしているか**: 既に提出した **movie_title_info PCA16** と **movie_title_info_large PCA16** の 2 本の CSV を読み、**ID でマージ**して target を 0.5:0.5 で平均し、1 本の提出にする。
- **効果**: 1 位と 2 位の設定の「情報の平均」で、片方の悪化をもう片方で補う効果を狙う。
- **注意**: 2 本の CSV が同じ test 行順で作られていること。本実装では test の ID 順でマージし直しているので、行順は test に一致する。

### 3.4 truncated_svd（追加実験としての 1 本）

- **何をしているか**: 次元削減を **TruncatedSVD 16** にした 1 本。§3 で 8 手法を全部回さずに、PCA 以外を 1 つだけ試す用。
- **効果**: PCA と同様に線形だが中心化しないので、embedding のスケールによっては SVD の方が合うことがある。

## 4. 本実験での実装上の注意（まとめ）

- **リーク防止**: すべての次元削減で **train だけで fit** し、test は **transform のみ**。
- **NMF**: embedding を train の min/max で [0, 1] にしたあと fit/transform。test も同じ min/max でスケール。
- **NaN/Inf**: 次元削減の出力に NaN/Inf が出た場合（UMAP や Kernel PCA で稀にある）、`lib/embedding_reduction.py` で **train の列中央値**で補正している。
- **提出前**: `lib/submission.save_submission` で予測を [0, 1] に clip し、NaN/Inf を 0.5 に置換してから CSV 保存。`verify_submission` で行数・列名・ID 一致・target 範囲を検証している。
